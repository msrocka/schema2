CalculationSetup

Type: object

A setup for a product system calculation.

root calculationType
Type: enum (of string)

The type of calculation that should be performed.

Must be one of:

  • "SIMPLE_CALCULATION"
  • "CONTRIBUTION_ANALYSIS"
  • "UPSTREAM_ANALYSIS"
  • "MONTE_CARLO_SIMULATION"

root productSystem
Type: object

The product system that should be calculated.

root productSystem @id
Type: string

The ID of the Ref.

root productSystem @type
Type: const
Specific value: "Ref"

root productSystem name
Type: string

The name of the entity.

root productSystem description
Type: string

The description of the entity.

root productSystem version
Type: string

A version number in MAJOR.MINOR.PATCH format where the MINOR and PATCH fields are optional and the fields may have leading zeros (so 01.00.00 is the same as 1.0.0 or 1).

root productSystem lastChange
Type: string

The timestamp when the entity was changed the last time.

root productSystem categoryPath
Type: array

The full path of the category of the referenced entity from top to bottom, e.g. "Elementary flows", "Emissions to air", "unspecified".

root productSystem library
Type: string

If the entity that is described by this reference is part of a library, this field contains the identifier of that library. The identifier is typically just the combination of the library name and version.

root productSystem refUnit
Type: string

This field is only valid for references of flows or impact categories and contains the name (symbol) of the reference unit of that respective flow or impact category.

root productSystem location
Type: string

This field is only valid for references of processes or flows and contains the location name or code of that respective process or flow.

root productSystem flowType
Type: enum (of string)

In case of a reference to a flow, this field can contain the type of flow that is referenced.

Must be one of:

  • "ELEMENTARY_FLOW"
  • "PRODUCT_FLOW"
  • "WASTE_FLOW"

root productSystem processType
Type: enum (of string)

In case of a reference to a process, this fiel can contain the type of process that is referenced.

Must be one of:

  • "LCI_RESULT"
  • "UNIT_PROCESS"

root process
Type: object

The process that should be calculated.

Same definition as productSystem

root impactMethod
Type: object

The LCIA method for the calculation.

Same definition as productSystem

root nwSet
Type: object

The normalisation and weighting set for the calculation.

Same definition as productSystem

root parameters
Type: array

A list of parameter redefinitions that should be used in the calculation.

root allocation
Type: enum (of string)

The calculation type to be used in the calculation.

Must be one of:

  • "PHYSICAL_ALLOCATION"
  • "ECONOMIC_ALLOCATION"
  • "CAUSAL_ALLOCATION"
  • "USE_DEFAULT_ALLOCATION"
  • "NO_ALLOCATION"

root withCosts
Type: boolean

Indicates whether life cycle costs should be also calculated.

root withRegionalization
Type: boolean

Indicates whether a regionalized result should be calculated or not.

root unit
Type: object

A Ref is a reference to a [RootEntity]. When serializing an entity (e.g. a [Process]) that references another standalone entity (e.g. a [Flow] in an [Exchange]) we do not want to write the complete referenced entity into the serialized JSON object but just a reference. However, the reference contains some meta-data like name, category path etc. that are useful to display.

Same definition as productSystem

root amount
Type: number

root flowProperty
Type: object

A Ref is a reference to a [RootEntity]. When serializing an entity (e.g. a [Process]) that references another standalone entity (e.g. a [Flow] in an [Exchange]) we do not want to write the complete referenced entity into the serialized JSON object but just a reference. However, the reference contains some meta-data like name, category path etc. that are useful to display.

Same definition as productSystem

root numberOfRuns
Type: integer

This field is only valid when this setup describes a Monte Carlo simulation and contains the number of simulation runs in that case.
